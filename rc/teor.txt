Отладка - этап разработки компьютерной программы, на котором обнаруживают, локализуют и
устраняют ошибки

Строка - тип данных, значениями которого является произвольна последовательность символов.
Обычно реализуется как массив символов

Хэш-функция - функция, осуществляющая преобразование массива входных данных произвольной
длины в выходную битовую последовательность установленной длины по определённому алгоритму

Функциональное программирование - парадигма, в которой процесс вычисления трактуется как
вычисление значений функций в их математическом понимании

Функция - это отдельный блок программы, который выполняет одно конкретное действие

Именованные параметры - это параметры, у которых есть имена, они передаются в функцию с
указанием имени параметра.

Функция высшего порядка - функция, принимающая или возвращающая другие функции

Функция первого порядка - функция, принимающая значения не функциональных типов данных и
возвращает значения таких же типов в качестве результата

Замыкание - функция первого порядка, в теле которой присутствуют ссылки на переменные,
объявленные вне тела этой функции в окружающем коде и не являющиеся е параметрами

Позиционные аргументы - значения, которые передаются в функцию в том порядке, в котором они
указаны

Простая рекурсия - вызов подпрограммы из неё же самое непосредственно

Косвенная рекурсия - вызов подпрограммы из неё же самой через другие подпрограммы

Подпрограмма - это поименованная или иным образом идентифицированная отдельная
функционально независимая часть компьютерной программы

Фактические параметры - это параметры, которые передаются в подпрограмму при её вызове

Формальные параметры - это параметры, которые объявлены при описании подпрограммы

Область видимости - часть программы, в пределах которой идентификатор остаётся связан с
сущностью, который он был назначен при объявлении

Пространство имён - множество уникальных идентификаторов (имён)

Линейная рекурсия - в теле функции присутствует только один вызов самой себя

Нелинейная рекурсия - в теле присутствует несколько вызовов самой себя

Головная рекурсия - вызов ближе к началу тела функции, хвостовая - другое

Синтаксические ошибки - ошибки интерпретации исходного текста программы при её запуске
(компиляции). Наличие синтаксических ошибок не позволит программе запуститься.

Ошибки времени выполнения - во время выполнения программы (деление на 0, некорректное
обращение к типам данных и тд)

Ленивые вычисления - применяемые в некоторых языках программирования стратегия, согласно
которой вычисления следует откладывать до тех пор, пока не понадобится их результат

Файл - поименованное место на носителе данных

Файловый дескриптор - целое число, которое присваивается операционной системой каждому потоку
ввода-вывода при его создании

Сигнал - это функция операционной системы, которая позволяет уведомлять программу о событии и
обрабатывать его асинхронно.

Декомпозиция - разделение задачи на множество частных задач, не превосходящих по совокупной
сложности исходную

Регулярные выражение - формальный язык поиска и модификации подстрок в тексте, основанный на
использовании метасимволов
